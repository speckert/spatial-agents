Causal Reasoning

Beyond pattern detection, Spatial Agents builds structural causal models that explain why events happen — using Judea Pearl's SCM framework and do-calculus for counterfactual reasoning.

Step 1: Event Detection

Behavioral patterns are detected from live position data. Each detected event carries a confidence score, the entities involved, the H3 cell where it occurred, and supporting metrics.

Step 2: DAG Construction

Detected events become nodes in a directed acyclic graph. Edges are discovered by matching event pairs against a domain knowledge rule base that encodes known causal relationships — for example, "adverse weather causes vessel loitering."

Edge Strength Calculation

strength = base_strength × temporal_decay × confidence_factor

Temporal decay penalizes events that are far apart in time (max gap: 6 hours). Confidence factor is the geometric mean of both events' confidence scores. Edges below strength 0.1 are discarded.

The graph is verified for acyclicity after each edge addition using NetworkX. This ensures the causal structure is always a valid DAG suitable for do-calculus operations.

Step 3: Counterfactual Interventions

Pearl's do-operator enables "what if" reasoning. Given a causal graph, the intervention engine can answer questions like: "If the weather event had not occurred, would the loitering still have happened?"

How It Works

1. Mutilate the graph — remove all incoming edges to the intervention target

2. Force the target node to a counterfactual value (typically zero)

3. Propagate effects through all descendants along causal paths

4. Report which downstream events would be affected and by how much

Step 4: FM Narration

The causal graph is serialized into a compact, token-efficient format and passed to the on-device Foundation Model. The FM generates a natural language narrative explaining the causal chain — turning graph structure into human-readable intelligence.
